4. Propuesta del Sistema para el Proceso Organizacional

4.1 Descripción del Problema

La empresa IceCold es una organización dedicada a la distribución y elaboración de helados personalizados la cual tiene como objetivo llegar a ser de las empresas más grandes a nivel nacional e internacional, siempre generando la mayor satisfacción a los clientes y mejorando sus servicios cada día.

El desafío estratégico al que se enfrenta IceCold radica en su elevado volumen de clientela, contrastado con un sistema de registro de usuarios y gestión de ventas excesivamente rudimentario, lo cual limita su escalabilidad y profesionalización.

Mantener un modelo rudimentario no solo frena el crecimiento, sino que erosiona ventajas competitivas en el largo plazo. La inversión en tecnología no es un gasto, sino un enabler para consolidar el posicionamiento de IceCold en el mercado.

4.2 Descripción del Sistema Propuesto

El sistema IceCold Web es una plataforma de gestión de pedidos y e-commerce, diseñada para optimizar los procesos organizacionales de la heladería.

El sistema está diseñado para necesidades cambiantes permitiendo agregar nuevas funcionalidades, soportando aumento concurrente de clientes, teniendo una interfaz agradable y adaptable según temporadas del año, funciona en todos tipos de plataformas (PC, Tablet, Móvil).

Siendo compatibles con cualquier tipo de sistemas existentes, como:

  • Dispositivos de navegadores (Safari, Edge, Firefox, etc)
  • Herramientas de gestión y pasarelas de pago

Garantizando una operación estable y segura con:

  • Encriptación de las transacciones
  • Copias de seguridad en la nube (backup automático)
  • Soporte y mantenimiento con un monitoreo 24/7 con alertas en tiempo real
  • Actualizaciones automáticas sin interrupciones

4.3 Funciones del Sistema Propuesto

1. Gestión Integral de Usuarios

Registro y verificación de identidad mediante validación multicapa (email, teléfono, huella digital). Clasificación automatizada de perfiles (ej.: clientes frecuentes, proveedores, administradores).

2. Gestión de Clientes B2B y B2C

Registro diferenciado para distribuidores (pedidos masivos) y clientes finales (pedidos personalizados), con validación de documentos (RUC para empresas).

3. Cálculo Dinámico de Métricas

Procesamiento en tiempo real de calificaciones, promedios ponderados y KPIs de desempeño (ej.: satisfacción del cliente).

4. Actualización en Tiempo Real de Inventario

Sincronización automática de stock con alertas de reposición y caducidad de productos.

5. Motor de Búsqueda Avanzada

Consultas multicriterio (por categoría, precio, ubicación) con filtros inteligentes y sugerencias predictivas.

6. Carrito de Compras y Personalización

Selección de productos, personalización de sabores/toppings/tamaño, gestión del carrito y confirmación de pedidos.

7. Sistema de Pedidos y Seguimiento

Registro de pedidos con número de orden, seguimiento de estados (pendiente, preparación, en camino, entregado) e historial de pedidos.

8. Notificaciones Automáticas

Alertas por cambio de estado de pedidos, promociones y novedades, con configuración personalizada de notificaciones.

9. Gestión de Inventario y Stock

Actualización automática de unidades disponibles, alertas de bajo stock y panel de reposición para administradores.

10. Panel Administrativo con Estadísticas

Visualización de datos de ventas, productos más vendidos, horarios pico y herramientas para toma de decisiones.

11. Sistema de Pago Integrado

Integración con pasarelas de pago (PayU, MercadoPago), soporte para tarjetas, transferencias y PSE, con validación automática de pagos.

4.4 Recursos Técnicos Requeridos

Infraestructura de Desarrollo

  • Backend: Java (Spring Boot/Servlets/JSP) + Node.js (Express.js)
  • Servidor: Apache Tomcat (Java) o servidor Node.js
  • Bases de datos: Oracle (producción), MySQL/PostgreSQL (pruebas), MongoDB Atlas (NoSQL)
  • Frontend: React.js, HTML5, CSS3 (Tailwind/Bootstrap), JavaScript (ES6+)

Entornos y Herramientas

  • IDEs: IntelliJ IDEA/Eclipse (Java), VS Code (frontend/Node.js)
  • Control de Versiones: Git + GitHub/GitLab
  • Pruebas de APIs: Postman o Insomnia
  • Diseño UI/UX: Figma + Canva

Servidores y Despliegue

  • Local: XAMPP/WAMP, Docker (opcional)
  • Producción: Vercel/Netlify (frontend), Railway/Heroku (backend)
  • Base de datos: Oracle Cloud, MongoDB Atlas

Seguridad y Dominio

  • Certificado SSL: Let's Encrypt (HTTPS)
  • Dominio: icecoldhelados.com
  • Correo Empresarial: Google Workspace/Zoho Mail
